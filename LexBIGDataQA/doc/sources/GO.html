LexBIG Data QA System - Gene Ontology

Documentation specifically relating to the QA of Gene Ontology (GO) by the LexBIG Data QA System.

Overview

This page covers specific information relating to the QA processes used to validate editions of the Gene Ontology (GO).

Details

Three key topics are covered here:
  • Data
  • Script Development
  • GO-specific QA Rules
GO Data

The original native-file data for GO is downloaded from the standard download website (see references section). We use the "Monthly releases" download (found under the v1.0 format section).  Despite being found under the v1.0 format section, the data actually appears in v1.2 format.

The LexBIG OBO loader is used to load the obo file into the LexBIG database with no modifications.

Analysis GO involves the following data states:
  1.    v1.2 obo file downloaded from GO website
  2.    LexBIG database loaded with (A) via LexBIG OBO loader
  3.    LexBIG API with loaded GO data from (a)

Script Development

Generating counts for OBO can make use of reusable utilities:
  • $QA_HOME/bin/countObo.pl for counting state A
  • $QA_HOME/bin/countDb.pl for counting state B
  • $QA_HOME/bin/countApi.pl for counting state C

The following tools were specifically developed for GO:
  • $QA_HOME/bin/rulesOboGO.pl
  • $QA_HOME/bin/rulesDbGO.pl
  • $QA_HOME/bin/rulesApiGO.pl
$QA_HOME/bin/rulesOboGO.pl
Input: <GO obo file>
Output: standard formatted rules file
Approximate Run Time: ~10 seconds
Assumptions:
  • n/a
Sample Call:
% cd $QA_HOME
% bin/GO/rulesGoObo.pl data/GO/20080901/gene_ontology_edit.obo >&! data/GO20080901/rulesA.txt
Notes:
This script calls the common $QA_HOME/bin/rulesObo.pl script and adds the results of GO-specific QA rules to it

GO-specific QA Rules

The following additional QA rules are needed for this terminology:

Domain Object
 Rule Name
 Rule Type
 Description
codedEntry
 codeHasDef
 content
 This rule passes if each non-obsolete concept has a definition.
codedEntry
 obsoleteCodeHasObsoleteDef
 content
 This rule passes if each obsolete concept has an OBSOLETE definition marker.
